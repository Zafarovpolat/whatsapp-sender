# WhatsApp Sender

Веб-платформа для массовой рассылки сообщений через WhatsApp. Запускает несколько WhatsApp-аккаунтов в headless-браузере (Puppeteer), управляет сессиями, проверяет номера и прогревает аккаунты.

---

## Стек технологий

| Слой         | Технологии                                                               |
| ------------ | ------------------------------------------------------------------------ |
| **Backend**  | Node.js, Express, Socket.IO, whatsapp-web.js, Puppeteer                  |
| **Frontend** | React (Vite), React Router, Axios, socket.io-client, qrcode.react        |
| **Деплой**   | Railway (Docker), клиент собирается в `client/dist` и раздаётся сервером |

## Как работает авторизация

Если переменная окружения `AUTH_PASSWORD` задана — вся API (`/api/*`) защищена токеном.

1. Фронтенд обращается к `GET /api/auth/check` — узнаёт, нужен ли пароль.
2. Если нужен — показывается `Login.jsx`, пользователь вводит пароль.
3. `POST /api/auth/login` → сервер проверяет пароль и возвращает токен (hex-строка 64 символа).
4. Токен сохраняется в `localStorage` и добавляется в каждый запрос через заголовок `x-auth-token`.
5. Тот же токен передаётся в Socket.IO через `auth: { token }`.

Если `AUTH_PASSWORD` **не задан** — авторизация отключена, все запросы проходят свободно.

## REST API

### Аутентификация

| Метод  | Путь              | Описание                        |
| ------ | ----------------- | ------------------------------- |
| `GET`  | `/api/auth/check` | Проверить, нужна ли авторизация |
| `POST` | `/api/auth/login` | Войти, получить токен           |

### Сессии (аккаунты)

| Метод    | Путь                | Описание                                 |
| -------- | ------------------- | ---------------------------------------- |
| `GET`    | `/api/sessions`     | Список всех сессий                       |
| `POST`   | `/api/sessions`     | Создать новый аккаунт `{ name, proxy? }` |
| `DELETE` | `/api/sessions/:id` | Удалить аккаунт                          |

### Файлы

| Метод  | Путь                      | Описание                                     |
| ------ | ------------------------- | -------------------------------------------- |
| `POST` | `/api/upload`             | Загрузить `.txt`-файл с номерами (multipart) |
| `GET`  | `/api/download/:filename` | Скачать результат чекера                     |

### Рассылка

| Метод  | Путь                | Описание            |
| ------ | ------------------- | ------------------- |
| `POST` | `/api/sender/start` | Запустить рассылку  |
| `POST` | `/api/sender/stop`  | Остановить рассылку |

**Параметры `/api/sender/start`:**

```json
{
  "sessionIds": ["id1", "id2"],
  "numbersFilePath": "/abs/path/to/numbers.txt",
  "messageTemplate": "{Привет|Здравствуйте}! Как дела?",
  "msgsPerAccount": 50,
  "totalMessages": 0,
  "typingDelayMin": 5,
  "typingDelayMax": 10,
  "pauseAfterMsgs": 10,
  "pauseDurationMin": 30,
  "pauseDurationMax": 60
}
```

### Чекер

| Метод  | Путь                 | Описание                                                 |
| ------ | -------------------- | -------------------------------------------------------- |
| `POST` | `/api/checker/start` | Начать проверку номеров `{ sessionId, numbersFilePath }` |
| `POST` | `/api/checker/stop`  | Остановить                                               |

### Прогрев

| Метод  | Путь                | Описание                                                                 |
| ------ | ------------------- | ------------------------------------------------------------------------ |
| `POST` | `/api/warmer/start` | Запустить прогрев (те же параметры, что у sender, кроме messageTemplate) |
| `POST` | `/api/warmer/stop`  | Остановить                                                               |

### Статус

| Метод | Путь          | Описание                                            |
| ----- | ------------- | --------------------------------------------------- |
| `GET` | `/api/status` | `{ sender, checker, warmer }` — что сейчас запущено |

## Socket.IO события

### Сервер → Клиент

| Событие            | Данные                         | Описание                                 |
| ------------------ | ------------------------------ | ---------------------------------------- |
| `sessions:update`  | `Session[]`                    | Обновлённый список сессий                |
| `sender:log`       | `string`                       | Строка лога рассылки                     |
| `sender:progress`  | `{ sent, remaining, account }` | Прогресс рассылки                        |
| `sender:complete`  | `{ totalSent }`                | Рассылка завершена                       |
| `sender:error`     | `string`                       | Ошибка рассылки                          |
| `checker:log`      | `string`                       | Строка лога чекера                       |
| `checker:progress` | `{ checked, total, valid }`    | Прогресс чекера                          |
| `checker:complete` | `{ total, valid, filename }`   | Чекер завершён, имя файла для скачивания |
| `checker:error`    | `string`                       | Ошибка чекера                            |
| `warmer:log`       | `string`                       | Строка лога прогрева                     |
| `warmer:progress`  | `{ sent, remaining }`          | Прогресс прогрева                        |
| `warmer:complete`  | `{ totalSent }`                | Прогрев завершён                         |

### Статусы сессии

| Статус          | Значение                        |
| --------------- | ------------------------------- |
| `initializing`  | Браузер запускается             |
| `qr`            | QR-код готов к сканированию     |
| `authenticated` | QR отсканирован, идёт загрузка  |
| `ready`         | Аккаунт онлайн и готов к работе |
| `disconnected`  | Потеряно соединение             |
| `auth_failure`  | Ошибка авторизации              |

## Запуск локально

```bash
# 1. Установить зависимости сервера
cd server && npm install

# 2. Установить зависимости клиента
cd ../client && npm install

# 3. Собрать клиент
npm run build

# 4. Запустить сервер (из корня или папки server)
cd ../server && node index.js
# или с автоперезагрузкой:
npx nodemon index.js
```

Откройте `http://localhost:3001` в браузере.

> В режиме разработки (без сборки) запустите `npm run dev` в папке `client` — Vite будет проксировать API-запросы на `http://localhost:3001`.

## Деплой на Railway

1. Подключите репозиторий к Railway.
2. Задайте переменную окружения `AUTH_PASSWORD`.
3. Railway автоматически использует `Dockerfile` из корня.
4. Сборка: клиент компилируется в `client/dist`, сервер раздаёт его как статику.

**Dockerfile-flow:**

- Сборка клиента: `npm run build` в `client/`
- Запуск сервера: `node server/index.js`

## Функциональность страниц

### Аккаунты (`/`)

- Добавление нового WhatsApp-аккаунта (имя + опциональный прокси)
- QR-код для сканирования с таймером (90 секунд)
- Карточки аккаунтов с цветными аватарами, статусами и номером телефона
- Удаление аккаунтов
- Skeleton-карточки при загрузке
- Реал-тайм обновления через Socket.IO

### Рассылка (`/sender`)

- Выбор нескольких аккаунтов-отправителей (только `ready`)
- Загрузка `.txt` файла с номерами (один номер на строку)
- Шаблон сообщения со **спинтаксом**: `{вариант1|вариант2}`
- Настройки: лимит сообщений с аккаунта, общий лимит, задержка имитации печати, паузы
- Прогресс-бар и журнал в реал-тайме

### Чекер (`/checker`)

- Проверяет список номеров — зарегистрированы ли они в WhatsApp
- Результат сохраняется в файл, доступный для скачивания
- Прогресс: проверено / есть WA / всего

### Прогрев (`/warmer`)

- Отправляет случайные сообщения (из `randomMessages.js`) на номера из файла
- Используется для «разогрева» новых аккаунтов перед массовой рассылкой
- Те же настройки задержек и пауз, что у рассылки

## Известные проблемы и заметки

### 401 Unauthorized в консоли

Возникает, когда `AUTH_PASSWORD` задан, но:

- токен ещё не получен (клиент не успел выполнить `initSocket(token)` до первых запросов)
- токен протух или был сброшен

**Решение:** `initSocket` вызывается только после того, как `ready === true` и `token` установлен. Проверьте, что `axios.defaults.headers.common['x-auth-token']` выставляется до любых запросов к `/api/sessions`.

### Хранение сессий

Сессии сохраняются в `server/data/sessions.json`. При перезапуске сервера они восстанавливаются автоматически с задержкой 5 секунд между каждой сессией.

### Прокси

Прокси передаётся как аргумент `--proxy-server=<proxy>` в Puppeteer. Формат: `http://ip:port`.
